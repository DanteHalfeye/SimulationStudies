## Random Walks and Object-Oriented Programming

I downloaded visual studio code with p5.visual and live server plugins.

This study explores the implementation of a two-dimensional random walk algorithm as an introduction to object-oriented programming (OOP) and procedural animation. A random walk is a mathematical process in which an object moves by taking a sequence of random steps, making it a simple but powerful model for simulating natural phenomena such as particle diffusion, animal movement, and stochastic systems.

The implementation begins by designing a Walker class responsible for storing the object's position and defining its behavior. The walker is initialized at the center of the canvas and repeatedly executes two primary methods:

step() — randomly selects the next movement direction.
show() — renders the walker on the canvas.

The movement is determined by generating uniformly distributed random numbers, giving every possible direction an equal probability. Initially, the walker is restricted to four directions (up, down, left, and right), demonstrating the fundamentals of discrete random motion. The study then explores more advanced variations, including diagonal movement and continuous motion by allowing floating-point step sizes.

Beyond demonstrating randomness, this exercise introduces key object-oriented concepts such as encapsulation, constructors, object state, and class methods. The separation between the walker's internal data (its position) and its behavior (movement and rendering) provides a clean and extensible architecture for future simulations.

This study serves as a foundation for more sophisticated generative systems, where random walks can be expanded into biased walks, Perlin noise motion, particle systems, autonomous agents, and other procedural animation techniques used in interactive graphics and game development.

    const choice = floor(random(4));
    if (choice == 0) {
      this.x++;
    } else if (choice == 1) {
      this.x--;
    } else if (choice == 2) {
      this.y++;
    } else {
      this.y--;
    }
  }
}

### Objective

Explore how changing the rendering and movement of the original random walker affects the visual output.

### Changes

- Changed the background from **white** to **black** to better highlight the generated shapes.
- Replaced the `point()` drawing with `arc()` to create larger and more expressive marks.
- Made the size of each arc depend on the walker's current position using `x²` and `y²`.
- Replaced the fixed black stroke with a random grayscale value for each arc.
- Modified the movement probabilities so the walker moves to the **right** more often, creating a directional bias.
- Added two extra walker instances (`walker1` and `walker2`) for future experiments with multiple walkers.

### Observations

These small changes transformed the original random walk into a generative drawing system. Instead of leaving a trail of points, the walker now creates evolving patterns with changing sizes, tones, and a noticeable drift toward the right side of the canvas.

class Fish {

  constructor(x, y, c, s) {
    this.pos = createVector(x, y);
    this.vel = p5.Vector.random2D();
    this.speed = 2;
    this.size = s;
    this.col = c;
  }

  separate(fishes) {

    let force = createVector(0, 0);
    let count = 0;

    for (let other of fishes) {

      if (other === this) continue;

      let d = p5.Vector.dist(this.pos, other.pos);
      let minDist = (this.size + other.size) * 0.8;

      if (d > 0 && d < minDist) {

        let diff = p5.Vector.sub(this.pos, other.pos);

        diff.normalize();
        diff.div(d);

        force.add(diff);
        count++;

      }
    }

    if (count > 0) {

      force.div(count);
      force.setMag(0.8);

      this.vel.add(force);

    }
  }

  move() {

    this.pos.add(this.vel);

    if (this.pos.x < -this.size)
      this.pos.x = width + this.size;

    if (this.pos.x > width + this.size)
      this.pos.x = -this.size;

    let surface = surfaceY(this.pos.x);

    if (this.pos.y < surface + this.size) {
      this.pos.y = surface + this.size;
      this.vel.y = abs(this.vel.y);
    }

    if (this.pos.y > height - this.size) {
      this.pos.y = height - this.size;
      this.vel.y *= -1;
    }

  }

  display() {

    push();

    translate(this.pos.x, this.pos.y);
    rotate(this.vel.heading());

    noStroke();
    fill(this.col);

    ellipse(0, 0, this.size * 1.6, this.size);

    triangle(
      -this.size * 0.8,
      0,
      -this.size * 1.4,
      this.size * 0.5,
      -this.size * 1.4,
      -this.size * 0.5
    );

    fill(255);
    ellipse(this.size * 0.4, -2, 4);

    fill(0);
    ellipse(this.size * 0.5, -2, 2);

    pop();

  }

}

this.target = null;

      }

    }

  }

  this.separate(blackFish);

  this.move();

}

findPlanktonCluster() {

  let best = null;
  let bestCount = 0;

  for (let p of plankton) {

    let count = 0;

    for (let other of plankton) {

      if (dist(
        p.pos.x,
        p.pos.y,
        other.pos.x,
        other.pos.y
      ) < 40) {

        count++;

      }

    }

    if (
    count >= 3 &&
    count > bestCount &&
    p.assigned < 5
) {

      best = p;
      bestCount = count;

    }

  }

  return best;

}

class RedFish extends Fish {

  constructor(x, y) {
    super(x, y, color(255, 70, 70), 12);
    this.speed = 2.2;
  }

  update(blackFish) {

    if (blackFish.length == 0) {
        this.move();
        return;
    }


    // Find nearest school center

    let target = null;
    let closest = Infinity;


    for(let school of schools){

        let members = blackFish.filter(
    b => b.target == school.plankton
);


        if(members.length > 0){

            let center = createVector(0,0);

            for(let b of members){
                center.add(b.pos);
            }

            center.div(members.length);


            let d = dist(
                this.pos.x,
                this.pos.y,
                center.x,
                center.y
            );


            if(d < closest){
                closest = d;
                target = center;
            }

        }

    }


    // Chase school

    if(target){

        let desired = p5.Vector.sub(
            target,
            this.pos
        );

        desired.normalize();


        // shark uncertainty
        desired.rotate(
            randomGaussian(0,radians(10))
        );


        desired.setMag(this.speed);

        this.vel.lerp(
            desired,
            0.08
        );

    }



    // Stay near surface

    let desiredY = height * 0.3;

    this.vel.y +=
      (desiredY - this.pos.y) * 0.003;
      



    this.separate([
        ...blackFish,
        ...redFish
    ]);


    for(let b of blackFish){

    let d = p5.Vector.dist(
        this.pos,
        b.pos
    );


    if(d < 12){

        b.dead = true;

    }

}
    this.move();


}

}


function surfaceY(x) {
  return (
    height * 0.2 +
    sin(x * 0.02 + t) * 18 +
    sin(x * 0.05 + t * 1.3) * 8
  );
}

## CHANGES TO THE SHARKFEST SIMULATION

1.  Remove the BlueFish and HunterFish systems entirely.

2.  Add a Plankton class

-   Spawn near the bottom of the water.
-   Very slow random walk (or Perlin drift).
-   Spawn continuously every random number of seconds.
-   Remove plankton when eaten.

3.  Black Fish lifecycle State: Exploring

-   Enter from the left or right edge every random interval.
-   Wander using a random walk.

Transition: - If a fish detects a cluster of at least 3 plankton within
a small radius, it becomes Feeding.

State: Feeding - Nearby black fish gain an increased probability of
moving toward the feeder. - Once several fish arrive they form a
school. - School members use Gaussian offsets around the school center
while eating.

4.  Red Sharks

-   Do not exist at the beginning.
-   After schools have formed twice and 10 seconds have passed, spawn a
    patrol of red sharks.
-   Sharks remain mostly near the upper part of the water.
-   They attack nearby schools.

5.  Explorer Shark (blue)

-   Rarely (1% chance while a shark is near the upper patrol area) one
    red shark becomes an explorer.
-   Explorer performs Lévy-flight movement.
-   Searches for dense schools.
-   Eats four black fish.
-   Returns to the nearest shark.
-   Changes back into a normal red shark.

6.  Influence

-   Mouse does not attract animals.
-   Mouse changes the current.
-   Currents modify movement probabilities and plankton distribution.

7.  Spawn / Despawn

-   Any creature leaving the screen is removed.
-   Every random number of seconds a new creature of that type enters
    from one of the screen edges.

Suggested update order each frame:

1.  Draw water.
2.  Spawn plankton if timer expires.
3.  Update plankton.
4.  Spawn black fish.
5.  Update black fish.
6.  Check for plankton clusters.
7.  Form schools.
8.  Spawn sharks if conditions are met.
9.  Update sharks.
10. Update explorer shark.
11. Remove off-screen entities.
12. Spawn replacements.
